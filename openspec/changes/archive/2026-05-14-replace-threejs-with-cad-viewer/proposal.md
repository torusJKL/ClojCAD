## Why

The current viewport layer is hand-rolled Three.js — scene setup, camera, renderer, OrbitControls, mesh construction, render loop, and a Reagent layer panel that's defined but unmounted. This is 125 lines of manual code for what `three-cad-viewer` (bernhard-42/three-cad-viewer) provides as a complete, purpose-built CAD visualization component: viewer, controls, tree panel, clipping planes, measurement, toolbar, SSAO, and material editing — all out of the box. Adopting it eliminates maintenance of custom Three.js wiring and unlocks professional CAD viewer features for free.

## What Changes

- **BREAKING**: Replace the entire manual Three.js viewport layer (`scene.cljs`, `controls.cljs`, `render.cljs`, `mesh_builder.cljs`) with the `three-cad-viewer` library's `Viewer` + `Display` components
- **BREAKING**: Remove the Reagent `layer_panel.cljs` component — the library's built-in tree panel replaces it
- **BREAKING**: Replace direct `three` dependency with `three-cad-viewer` (which transitively provides `three@0.184.0`)
- Adapt the scene manager to maintain dual state: our `scene` atom for bookkeeping + `Viewer.addPart/removePart/updatePart` API for rendering
- Adapt the tessellation kernel to produce the library's full `Shape` format (add `obj_vertices`, `face_types`, `edge_types`, `triangles_per_face`, `segments_per_edge`)
- Map tagged sub-geometry (the `tag` macro) to hierarchical tree children in the library
- Wire the library's notification callback to sync tree panel state changes back to the scene atom
- Update `public/index.html` to load the library's CSS and use the new display container structure
- Remove `scripts/patch-three.js` (the Three.js exports map patch is no longer needed since we no longer shadow-cljs-require Three.js directly — the library consumes it internally via ESM)
- Simplify `core.cljs` init flow: `Display` + `Viewer` construction replaces `init-viewport!` + `init-controls!` + `start-loop!`

## Capabilities

### New Capabilities
- `cad-viewer-integration`: Integration of the three-cad-viewer library — Viewer/Display lifecycle, shape data adapter, tree panel sync, and configuration

### Modified Capabilities
- `interactive-viewer`: Requirements change from "manual Three.js viewport with OrbitControls" to "library-based viewer with tree panel, toolbar, clipping, measurement"
- `cad-geometry-kernel`: Tessellation output expands from 4-field map to full 9-field `Shape` format with face/edge classification metadata
